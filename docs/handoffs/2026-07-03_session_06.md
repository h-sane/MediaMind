## Architectural decisions

### OpenCV YuNet+SFace listed first in catalog (D1)
It is the only Apache-2.0 provider — commercially and personally usable without restriction. buffalo_l is non-commercial research only. Listing YuNet+SFace first makes the license story clearer: users see the free option before the restricted one. The UI shows both with their license badges; the user must accept the license before download.

### archive="direct" vs "zip" (D2)
Direct downloads skip the zip step: each file in `entry.downloads` is fetched to `extract_dir / filename`. This is the right model for providers that ship as individual ONNX files rather than a zip pack. The download runner already handled the zip case; the only fix was routing the destination path for direct files.

### PyInstaller one-dir mode (D3)
One-dir (`COLLECT` + `EXE(exclude_binaries=True)`) is chosen over one-file mode because:
- Faster startup (no temp extraction step on launch)
- Easier debugging (DLLs visible in `_internal/`)
- Better for electron-builder `extraResources` bundling (predictable folder layout)
The trade-off is a larger `resources/engine/` folder in the installer.

### SHA-256 pinned to None for now (D4)
Real sha256 hashes require downloading the ~38 MB files. They're pinned to `None` with a `TODO M8 final` note. The download will succeed but skip hash verification. Before the public release, the hashes must be pinned: download the files once, compute `sha256sum`, update catalog.py.

## Important implementation notes

- **cv2.FaceDetectorYN** requires `opencv-contrib-python >= 4.5.4`. Verified available in `C:\Users\husai\faces-env` (cv2 4.13.0). Standard `opencv-python` does NOT have these APIs.
- **YuNet detection format**: each detection row has 15 values: `[x, y, w, h, re_x, re_y, le_x, le_y, nose_x, nose_y, rcm_x, rcm_y, lcm_x, lcm_y, score]`. The bbox is `(x, y, x+w, y+h)` in absolute pixels.
- **SFace feature**: `recognizer.feature(aligned)` returns shape `(1, 128)`. Must flatten and L2-normalize before storing.
- **Packaged engine path**: `join(process.resourcesPath, 'engine', 'mediamind.exe')` on Windows. The `_internal/` subdirectory (from PyInstaller one-dir) sits alongside `mediamind.exe` inside `resources/engine/`.
- **electron-builder extraResources**: `from: "../backend/dist/mediamind"` is relative to `app/electron-builder.yml`. This assumes `backend/dist/mediamind/` exists before running `npm run dist`.
- **Build order for release**: (1) `cd backend && pyinstaller mediamind.spec` → (2) `cd app && npm run dist`
