## Architectural decisions

- **Electron + React UI, Python FastAPI backend over localhost HTTP** (user
  decision). Backend binds 127.0.0.1 on a free port, prints
  `MEDIAMIND_PORT=<port>`; Electron main passes a random session token via
  `MEDIAMIND_TOKEN`, sent as `X-MediaMind-Token` on every request.
- **Electron never touches user media** — one audited code path (backend
  `core/safety.py`) can modify files.
- **`FaceProvider.get_faces()` combines detect+embed** (diverges from the plan
  doc's detect→embed split) because every real backend computes both in one
  pass; a split interface would force useless state caching.
- **Multi-person media routes to `needs_review`** (PRD F3) instead of V0's
  copy-into-every-folder — see `core/organizer.py::route_media`.
- **Embedding cache keyed by (content hash, provider id)** with a "no faces
  found" sentinel row so analyzed-but-empty files also skip re-analysis.
- **Fake provider = color recognizer** (`providers/fake.py`): solid-color
  fixtures cluster like people, enabling full pipeline tests with no model.
- **Per-library SQLite in `<library>/.mediamind/index.db`** — travels with the
  folder; always rebuildable; filesystem stays the source of truth.
- **Apache-2.0** for code; model licenses surfaced per-provider (buffalo_l is
  non-commercial research).
