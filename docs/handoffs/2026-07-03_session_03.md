## Architectural decisions

### FaceRecord carries full spatial info (D1)
`extract_faces()` previously discarded `bbox`. Now every face has `(frame_no, bbox, embedding)` as `FaceRecord`. The `MediaFaces.embeddings` property provides back-compat for `clustering.py` without any change to that module.

### Embedding cache stores bbox (D2)
Schema v3 adds `frame_no, bbox_*` to `embeddings`. Pre-v3 rows with `frame_no IS NULL` return `None` from `get_cached_faces()` (= re-detect to acquire bbox). `get_cached()` back-compat wrapper bypasses this check since old callers don't need bbox.

### faces rows carry own embedding BLOB (D3)
Rather than FK into `embeddings`, `faces` stores its own `embedding BLOB`. This avoids fragility when `put_cached_faces` does DELETE+INSERT (invalidating any rowid FK). Cost: ~2 KB per face, negligible. Enables self-contained centroid recomputation in `merge_persons`.

### Persons reconciliation by centroid similarity (D4)
Every face scan reclusters all embeddings (detection is the expensive part; clustering is cheap). New cluster centroids are greedy-matched to existing person centroids (cosine sim ≥ MATCH_SIM_THRESHOLD = 0.5 = 1 − DEFAULT_EPS). Matched → user name survives. Unmatched cluster → new Person_NNN. Named person with no matching cluster → kept (0 faces). Unnamed → deleted.

### Downloads are `__app__`-scoped jobs (D5)
`JobManager` is library-keyed; provider downloads use `library_id="__app__"`. No conflicts with library scans. `selectJobByType` finds the download job by type across all jobs.

### Static Python catalog instead of JSON resource (D6)
Avoids PyInstaller resource collection complexity. Catalog is a list of `CatalogEntry` dataclasses in `providers/catalog.py`. New providers need only a catalog entry + provider class (or just a manifest JSON for future extensibility).

### `ProviderManager` injectable in `create_app()` (D7)
`create_app(provider_manager=ProviderManager(tmp_path, catalog=[fake_entry]))` in tests = no disk I/O, no real downloads. Production: `create_app()` creates `ProviderManager(models_dir())` in lifespan.

## Problems encountered and solutions

1. **Migration test hard-coded "2"**: `test_v1_db_migrates_to_v2` asserted version == "2"; now schema is v3. Fixed to `str(SCHEMA_VERSION)` and updated assertions to check v3 tables.

2. **`get_cached` back-compat**: `get_cached` wrapper called `get_cached_faces` which returns `None` for NULL frame_no rows. But `put_cached` inserts rows without frame_no. Fixed: `get_cached` queries the DB directly, bypassing the NULL frame_no check.

3. **`extract_file_faces` — `_frames_for` takes `ScannedFile` but `load_frame` passes a Path**: Used an `_Stub` class inside `load_frame` that has the `.path` and `.kind` attributes expected by `_frames_for`. Clean; avoids importing `ScannedFile` into the type signature.

4. **`ScanProgress` hardcoded dedupe result display**: The component showed `job.result.groups` / `job.result.reclaimable_bytes` for any succeeded job. Fixed to check `job.type` and show appropriate text for faces vs. dedupe.

5. **Radix UI not installed**: `ProvidersScreen` originally imported `@radix-ui/react-dialog`. Radix is not in package.json. Rewrote the dialog as a plain overlay (same pattern as `ConfirmTrashDialog` in `DedupeReview`). No new dependencies added.
